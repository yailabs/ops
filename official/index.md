# YAI Official Document Set

Purpose: provide a procurement- and funding-ready document set anchored to verifiable evidence.

Read order:
1) Overview — what YAI is, what is proven today, what is next
2) Industrial Plan — TRL roadmap + execution plan + budget narrative
3) Technology Dossier — architecture + runtime grammar + moat
4) Assurance Statement — verification, limits, risk controls, audit posture
5) Pilot Delivery Spec — scope boundaries + DoD + timeline + requirements
6) Legal Pack — pilot SOW + commercial/legal notes

All claims must be traceable to an Evidence Baseline (wave_id + hashes).
