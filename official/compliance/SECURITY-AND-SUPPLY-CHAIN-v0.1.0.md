# SECURITY & SUPPLY CHAIN (v0.1.0)

## Purpose
Baseline security posture + software supply chain integrity mapping.

## Controls baseline
- ISO/IEC 27001/27002 posture mapping (high level)
- Operational hardening: least privilege, egress controls, secrets handling
- Logging & monitoring (OTEL where applicable)

## Supply chain
- NIST SSDF mapping to repo/CI practices
- SLSA provenance expectations for releases/waves
- Vulnerability intake + handling process (disclosure, patch, advisory)

## CRA overlay (when applicable)
Declare triggers and required evidence for CRA-aligned posture.
