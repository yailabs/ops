# Pilot SOW v1.0 (SC102-WAVE1-LAUNCH)

## 1. Purpose

This Statement of Work defines a 14-day pilot to validate YAI runtime governance controls on one production-relevant workflow.

## 2. Scope

- Launch reference: `SC102-WAVE1-LAUNCH`
- Baseline proof package: SC102 Wave 1 (`D1 + D8`)
- One customer workflow mapped to the Star Case: AI Production Change Guard

Out of scope:

- full enterprise rollout
- all-domain expansion (D1->D9)
- custom UI or unrelated platform refactors

## 3. Deliverables

1. Runtime governance setup and health validation.
2. Wave execution result and evidence bundle.
3. Verification output and KPI summary.
4. Operator demo walkthrough.
5. Expansion recommendation (go / no-go).

## 4. Timeline (14 Days)

- Days 1-2: discovery, workflow mapping, acceptance alignment
- Days 3-6: environment setup and baseline runs
- Days 7-10: customer workflow adaptation and reruns
- Days 11-13: reporting, security/compliance review
- Day 14: executive decision session

## 5. Responsibilities

YAI team:

- pilot execution
- evidence generation and verification
- report and recommendation

Customer team:

- provide environment access and owners
- define workflow and policy constraints
- attend checkpoints and decision session

## 6. Definition of Done

Pilot is complete only if all are true:

- wave execution passes
- bundle verification passes
- prohibited effects are blocked per contract baseline
- stakeholders receive report with explicit go/no-go recommendation

## 7. Commercial Terms (Template)

- Pilot fee: fixed fee (to be agreed)
- Payment terms: to be agreed
- Conversion credit: optional (to be agreed)

## 8. Exit Criteria

- Expand: approved annual scope and rollout plan
- Stop: pilot closed with documented findings and no further obligations
